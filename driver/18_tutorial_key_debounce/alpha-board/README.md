# 教程 18: 带消抖的按键驱动（中断 + 工作队列）

## 概述

本驱动使用中断和工作队列实现**正确消抖**的按键驱动。它展示了如何处理真实世界机械开关存在的抖动问题。

## 主要特性

- **字符设备接口** `/dev/imxaes_key_debounce`
- **中断驱动**（双边沿触发）
- **工作队列消抖**：20ms 延迟等待抖动稳定
- **状态比较**：只在状态真正变化时报告事件
- **统计信息**：IRQ 计数、事件计数、跳过的抖动
- **Poll 支持**：非阻塞 I/O
- **硬件抽象**：使用 `key_hw.c` 进行 GPIO/IRQ 操作

## 硬件连接

- **GPIO**: GPIO1_IO18 (UART1_CTS_B 引脚)
- **有效电平**: 低电平（按键按下拉低 GPIO）
- **中断**: 上升沿和下降沿

## 设备树

```dts
pinctrl_key: keygrp {
    fsl,pins = <
        MX6UL_PAD_UART1_CTS_B__GPIO1_IO18  0x1b0b0
    >;
};

imxaes_key_debounce: key-debounce {
    compatible = "imxaes-key-debounce";
    pinctrl-names = "default";
    pinctrl-0 = <&pinctrl_key>;
    gpios = <&gpio1 18 GPIO_ACTIVE_LOW>;
    status = "okay";
};
```

## 编译

```bash
cd driver/18_tutorial_key_debounce/alpha-board
make
```

输出: `out/driver_artifacts/18_tutorial_key_debounce/18_tutorial_key_debounce_driver.ko`

## 使用方法

### 加载驱动
```bash
insmod 18_tutorial_key_debounce_driver.ko
```

### 检查设备节点
```bash
ls -l /dev/imxaes_key_debounce
```

### 使用应用测试
```bash
./key_demo debounce
```

**预期行为**：每次按键只产生一个事件，感谢消抖处理。

### 卸载驱动
```bash
rmmod 18_tutorial_key_debounce_driver
```

卸载时会打印统计信息：
```
key_remove: statistics - IRQs: 42, events: 2, skipped: 38
```

## 消抖算法

```
┌─────────────────────────────────────────────────────────────┐
│                    机械开关抖动现象                           │
│                                                              │
│  GPIO: ────┐  ┌─┐ ┌──┐    ┌───────┐                        │
│           └──┘ └──┘         └───┘                           │
│           ↑  ↑↑↑  ↑↑↑  ↑    ↑                               │
│           └──┴────┴────┴────┘ 全部在约5-20ms内               │
│                                                              │
│  无消抖：检测到 5 个事件！                                   │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│                    使用消抖算法                              │
│                                                              │
│  1. 中断在边沿变化时触发                                     │
│  2. 调度工作队列                                             │
│  3. 等待 20ms（抖动稳定）                                    │
│  4. 读取稳定的 GPIO 状态                                     │
│  5. 与上次状态比较                                           │
│     - 如果变化：报告事件                                     │
│     - 如果相同：跳过（只是抖动）                             │
│                                                              │
│  结果：检测到 1 个事件！                                     │
└─────────────────────────────────────────────────────────────┘
```

## 与教程 17 对比

| 特性 | 教程 17 | 教程 18 |
|------|---------|---------|
| 检测方式 | 轮询（阻塞） | 中断 |
| CPU 占用 | 高（忙等待） | 低（事件驱动） |
| 消抖 | 无 | 20ms 工作队列 |
| 每次按键事件数 | 多次（抖动） | 一次（正确） |
| Poll 支持 | 无 | 有 |
| 统计信息 | 无 | 有 |
